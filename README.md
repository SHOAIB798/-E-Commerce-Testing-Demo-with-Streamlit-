
# E-Commerce Testing Demo with Streamlit

This project is a simple yet effective e-commerce testing demonstration built using Streamlit. The application focuses on core quality assurance concepts, allowing users to log in, select products, specify quantity, and calculate the total price. Through this application, key testing principles are demonstrated, including test case design, bug identification, and basic validation of user inputs.

### Features:

1. **User Login:** The application starts with a login screen where users must enter the correct username and password ("admin" and "1234"). Without correct credentials, users are not allowed to proceed. This ensures that test cases can cover authentication scenarios.

2. **Product Selection:** Upon successful login, users can select from a list of predefined products (e.g., Laptop, Phone, Headphones). Each product is associated with a specific price, which allows for total calculation.

3. **Quantity Input:** Users can specify the quantity of the selected product. This input is validated to ensure it’s a positive number.

4. **Cart Total Calculation:** Once a product and quantity are selected, users can calculate the total price. This demonstrates basic output validation and ensures that correct calculations are made.

### Testing Focus:

This project is designed with a strong emphasis on manual testing principles. By using this app, testers can design test cases for:

* Valid and invalid login attempts.
* Product selection edge cases (e.g., no product selected
